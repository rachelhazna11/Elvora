# Plan 06-08: Fix Checkout Validation — Required Province, Postal Code, Phone + Payment Fields

## Goal
Add required-field validation for Province, Postal Code, and Phone in checkout Step 1, and make the payment card fields editable with basic required validation in Step 3.

## Context

**Bug 1 — Step 1 missing required fields:**
`validateStep1()` checks email, firstName, lastName, address, city — but NOT province, postalCode, or phone. HTML labels have no `*` and no error bindings. User can skip these and advance.

**Bug 2 — Payment fields are readonly:**
Step 3 inputs have hardcoded values and `readonly`. Per CLAUDE.md ("Payments: Placeholder architecture only"), these should be editable so the user can type something, with simple non-empty validation (no real card format check needed).

UAT gaps: Gap 5 (Province/PostalCode/Phone not required), Gap 6 (card fields not required).

## Tasks

### Task 1: Add province, postalCode, phone validation to validateStep1()

**File:** `checkout.html`
**Action:** modify — inside the `validateStep1()` function in the `<script>` block

After `if (!this.city.trim()) this.errors.city = 'City required';`, add:
```js
        if (!this.province.trim()) this.errors.province = 'Province required';
        if (!this.postalCode.trim()) this.errors.postalCode = 'Postal code required';
        if (!this.phone.trim()) this.errors.phone = 'Phone number required';
```

### Task 2: Update Province label and add error binding

**File:** `checkout.html`
**Action:** modify — Province input in Step 1 HTML

Change label text from `Province` to `Province *` and add `:class` + error `<p>` binding below the input.

### Task 3: Update Postal Code label and add error binding

**File:** `checkout.html`
**Action:** modify — Postal Code input in Step 1 HTML

Change label text from `Postal Code` to `Postal Code *` and add `:class="{ 'error': errors.postalCode }"` to the input and an error paragraph below it.

### Task 4: Update Phone label and add error binding

**File:** `checkout.html`
**Action:** modify — Phone input in Step 1 HTML

Change label text from `Phone` to `Phone *` and add `:class="{ 'error': errors.phone }"` to the input and an error paragraph below it.

### Task 5: Add cardNumber, expiry, cvv state fields and validatePayment()

**File:** `checkout.html`
**Action:** modify — inside the `checkoutData()` function return object in the `<script>` block

Add after `errors: {}`:
```js
      cardNumber: '',
      expiry: '',
      cvv: '',
```

Add `validatePayment()` method before `nextStep()`:
```js
      validatePayment() {
        if (!this.cardNumber.trim()) { this.errorMsg = 'Please enter a card number.'; return false; }
        if (!this.expiry.trim()) { this.errorMsg = 'Please enter the expiry date.'; return false; }
        if (!this.cvv.trim()) { this.errorMsg = 'Please enter the CVV.'; return false; }
        this.errorMsg = '';
        return true;
      },
```

Call it at the start of `placeOrder()`:
```js
      async placeOrder() {
        if (!this.validatePayment()) return;
        this.loading = true;
```

### Task 6: Replace readonly card inputs with editable x-model inputs

**File:** `checkout.html`
**Action:** modify — Step 3 payment form HTML

Replace the three readonly inputs (Card Number, Expiry, CVV) with editable inputs bound via `x-model`:

Card Number: `x-model="cardNumber"`, placeholder `"1234 5678 9012 3456"`, `maxlength="19"`, remove `readonly` and static `value`.

Expiry: `x-model="expiry"`, placeholder `"MM/YY"`, `maxlength="5"`, remove `readonly` and static `value`.

CVV: `x-model="cvv"`, placeholder `"123"`, `maxlength="4"`, remove `readonly` and static `value`.

Also update the mock card visual number display:
BEFORE: `<p class="payment-card-number">•••• &nbsp;•••• &nbsp;•••• &nbsp;4242</p>`
AFTER: `<p class="payment-card-number" x-text="cardNumber || '•••• •••• •••• ••••'"></p>`
