# Plan 06-03: Mini Cart Drawer — Inject HTML + Wire Nav Icon + Wire product.html addToBag

## Goal
A slide-out mini cart drawer is visible on every page: clicking the nav cart icon opens it, adding a product to the bag from `product.html` also opens it, and the drawer correctly renders items with quantity steppers, subtotal, and a link to the full cart page.

## Context
Plan 06-02 added `cartDrawerOpen`, `openDrawer()`, and `closeDrawer()` to `Alpine.store('cart')` in `js/cart.js`. Plan 06-01 added all `.cart-drawer-*` CSS classes.

`js/components.js` injects the nav via `navRoot.innerHTML = navHTML` and calls `Alpine.initTree(navRoot)`. The cart icon in `navHTML` (around line 172 in components.js) is currently an `<a href="/cart.html">`. The mobile cart link at line 227 is also `<a href="/cart.html">`.

The cart drawer must be injected as a sibling to the nav (appended to `document.body`) and initialized with `Alpine.initTree`. It must NOT be placed inside `navRoot` because the nav has a `backdrop-filter` that would clip the drawer's `position: fixed` stacking.

`product.html` currently calls `Toastify({ text: 'Added to Bag', ... }).showToast()` after `Alpine.store('cart').add(...)` in `addToBag()` (around line 235). The Toastify notification is replaced by opening the drawer. Keep Toastify only for wishlist saves and error states.

`shop.html` quick-add button (around line 286) calls `Alpine.store('cart').add(...)` then `Toastify(...)`. Update it similarly.

## Tasks

**File:** `js/components.js`
**Action:** modify

**1a. Add the `cartDrawerHTML` constant** after the `footerHTML` constant (after line 314, before the nav/footer injection block). This is an Alpine x-data component. The `x-data` attribute must be an empty string or `{}` since the drawer reads from `$store.cart` — NOT from inline component state.

The drawer consists of two sibling elements inside a wrapper div: a backdrop div and a panel div.

**1b. Inject the drawer** into the DOM. After the existing nav/footer injection block (after line 328 in components.js where `Alpine.initTree(footerRoot)` is called), add:

```js
// ─── Cart drawer injection ────────────────────────────────────────────────────
const cartDrawerRoot = document.createElement('div');
cartDrawerRoot.innerHTML = cartDrawerHTML;
document.body.appendChild(cartDrawerRoot);
if (window.Alpine) Alpine.initTree(cartDrawerRoot);
```

### Task 2: Update `product.html` addToBag to open drawer

**File:** `product.html`
**Action:** modify

Find the `addToBag()` method in the inline `x-data` object. It currently calls `Alpine.store('cart').add(...)` then `Toastify({ text: 'Added to Bag', ... }).showToast()`.

Replace the Toastify "Added to Bag" call with a call to open the cart drawer:

```js
// BEFORE (the Toastify block after Alpine.store('cart').add()):
Toastify({
  text: 'Added to Bag',
  duration: 2500,
  gravity: 'bottom',
  position: 'right',
  style: {
    background: 'var(--charcoal)',
    color: '#ffffff',
    borderRadius: '50px',
    fontSize: '12px',
    letterSpacing: '2px',
  },
}).showToast();

// AFTER:
Alpine.store('cart').openDrawer();
```

The `addToBag()` method still validates `this.selectedSize` and calls `Alpine.store('cart').add(...)` unchanged — only the Toastify notification at the end is replaced with `Alpine.store('cart').openDrawer()`. Keep all other Toastify calls in `product.html` (wishlist saves, error messages) intact.

### Task 3: Update `shop.html` quick-add to open drawer

**File:** `shop.html`
