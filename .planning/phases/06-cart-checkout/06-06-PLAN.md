# Plan 06-06: Fix Dual Alpine Registration + Eager Cart Overwrite

## Goal
Eliminate the double `initAuth()` registration in `components.js` that causes `onAuthStateChange` to fire twice, and remove the eager `loadCartFromSupabase()` call that races with user-added items and wipes the cart wholesale.

## Context
Two bugs in `js/components.js` compound each other:

**Bug 1 — Dual registration (lines 49–51):**
Both the `alpine:init` listener AND the direct `if(window.Alpine)` branch call `registerStores()` and `initAuth()`. When Alpine has already initialized before the ES module runs (the common case with `defer` CDN), BOTH branches execute. This registers two `onAuthStateChange` subscriptions. When `SIGNED_IN` fires, `mergeGuestCartToSupabase()` runs twice; when `INITIAL_SESSION` fires, `loadCartFromSupabase()` runs twice — causing race conditions and double DOM mutations.

**Bug 2 — Eager loadCartFromSupabase (line 28):**
`initAuth()` eagerly calls `loadCartFromSupabase(user.id)` right after `getUser()`. This races against any items the user adds between page load and async response. When it resolves, `cart.js` does `Alpine.store('cart').items = items` — a wholesale replace wiping anything added since load. The `INITIAL_SESSION` event handler already handles this correctly; the eager call is redundant and harmful.

UAT gaps addressed: Gap 3 (cart items disappear after login), Gap 4 (product page shows raw code when logged in).

## Tasks

### Task 1: Add one-shot guard to components.js

**File:** `js/components.js`
**Action:** modify

Replace the dual-registration block at the bottom of the auth setup section with a guarded one-shot init:

BEFORE:
```js
// alpine:init path: fires if Alpine hasn't initialized yet (e.g. script order changes)
document.addEventListener('alpine:init', () => { registerStores(); initAuth(); });
// Direct path: Alpine has already initialized before this module ran
if (window.Alpine) { registerStores(); initAuth(); }
```

AFTER:
```js
// Guard: ensure registerStores/initAuth run exactly once regardless of Alpine init timing.
let _componentsInitDone = false;
function _componentsInit() {
  if (_componentsInitDone) return;
  _componentsInitDone = true;
  registerStores();
  initAuth();
}
document.addEventListener('alpine:init', _componentsInit);
if (window.Alpine) _componentsInit();
```

### Task 2: Remove eager loadCartFromSupabase from initAuth

**File:** `js/components.js`
**Action:** modify

Inside `initAuth()`, remove the eager `loadCartFromSupabase` call after `getUser()`. The `INITIAL_SESSION` event handler already loads the Supabase cart for returning authenticated users.

BEFORE (inside initAuth):
```js
  const user = await getUser();
  Alpine.store('auth').user = user;
  Alpine.store('auth').loggedIn = !!user;

  if (user) {
    loadCartFromSupabase(user.id).catch(err => console.warn('[cart] initial load failed:', err));
  }
```

AFTER:
```js
  const user = await getUser();
  Alpine.store('auth').user = user;
  Alpine.store('auth').loggedIn = !!user;
  // Cart load handled by INITIAL_SESSION event handler below — no eager call here.
```
