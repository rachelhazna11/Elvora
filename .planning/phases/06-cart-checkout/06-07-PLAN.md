# Plan 06-07: Google OAuth Profile Persistence + Guest Cart Merge on Redirect

## Goal
Add a Supabase DB trigger that creates a `user_profiles` row when a Google OAuth user signs in for the first time, and fix the `INITIAL_SESSION` cart handler so guest items are merged (not lost) after an OAuth redirect return.

## Context

**Bug 1 — No user_profiles trigger for Google OAuth:**
`signInWithOAuth()` delegates to Supabase but writes nothing to `user_profiles`. Profile rows are only created inside `account.html loadProfile()` — so Google users who land elsewhere first have no profile row. Fix: DB trigger `AFTER INSERT ON auth.users` reads `raw_user_meta_data` (set by Google) to insert the profile row.

**Bug 2 — Guest cart lost on OAuth redirect:**
Google OAuth is a redirect flow. After redirect return, Supabase fires `INITIAL_SESSION` (not `SIGNED_IN`). The `INITIAL_SESSION` branch only calls `loadCartFromSupabase`, which replaces the Alpine store with the (empty) Supabase cart, wiping local guest items. Fix: in `INITIAL_SESSION`, check for local items first — if present, merge them.

UAT gaps: Gap 1 (guest cart lost after Google sign-in), Gap 2 (Google OAuth profile not saved to Supabase).

## Tasks

### Task 1: Create Supabase migration for handle_new_user trigger

**File:** `supabase/migrations/005_handle_new_user.sql` (create new)
**Action:** create

Create the file with this content:

```sql
-- Migration 005: Auto-create user_profiles row on auth.users insert.
-- Applies to email/password signups AND Google OAuth (raw_user_meta_data contains given_name etc).

CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS trigger
LANGUAGE plpgsql
SECURITY DEFINER
SET search_path = public
AS $$
BEGIN
  INSERT INTO public.user_profiles (id, first_name, last_name, username)
  VALUES (
    NEW.id,
    COALESCE(NEW.raw_user_meta_data->>'given_name', NEW.raw_user_meta_data->>'first_name', split_part(COALESCE(NEW.email, ''), '@', 1)),
    COALESCE(NEW.raw_user_meta_data->>'family_name', NEW.raw_user_meta_data->>'last_name', ''),
    COALESCE(NEW.raw_user_meta_data->>'preferred_username', NEW.raw_user_meta_data->>'username', split_part(COALESCE(NEW.email, ''), '@', 1))
  )
  ON CONFLICT (id) DO NOTHING;
  RETURN NEW;
END;
$$;

DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users;
CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE FUNCTION public.handle_new_user();
```

This file is a reference — the user must run it in the Supabase SQL Editor (Dashboard → SQL Editor) since the project has no migration runner configured.

### Task 2: Fix INITIAL_SESSION to merge guest cart on OAuth redirect return

**File:** `js/components.js`
**Action:** modify

In the `onAuthChange` callback inside `initAuth()`, update the `INITIAL_SESSION` branch:

BEFORE:
```js
    } else if (user && event === 'INITIAL_SESSION') {
      try { await loadCartFromSupabase(user.id); } catch (err) { console.warn('[cart] load failed:', err); }
    }
```

AFTER:
```js
    } else if (user && event === 'INITIAL_SESSION') {
      const localItems = loadFromStorage();
      if (localItems.length > 0) {
        // OAuth redirect return may carry guest items — merge instead of overwrite.
        try { await mergeGuestCartToSupabase(user); } catch (err) { console.warn('[cart] merge on session restore failed:', err); }
      } else {
        try { await loadCartFromSupabase(user.id); } catch (err) { console.warn('[cart] load failed:', err); }
      }
    }
```
