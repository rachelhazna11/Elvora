# Plan 06-04: Full Cart Page — cart.html Implementation

## Goal
`cart.html` renders a fully functional cart page with an items list (quantity steppers, remove, line total), an order summary sidebar (subtotal, free shipping note, total, proceed CTA), and an empty state with a "Shop Now" CTA — all powered by `Alpine.store('cart')`.

## Context
`cart.html` currently has `<main><!-- Phase 6 fills this --></main>` as a stub. It already loads `js/cart.js`, `js/supabase.js`, and `js/components.js` via `<script type="module">` tags. Alpine.js is loaded via CDN defer script. `Alpine.store('cart')` will be initialized by `cart.js` before this page's `x-data` renders.

Plan 06-01 added all `.cart-page`, `.cart-layout`, `.cart-item-row`, `.cart-item-image`, `.order-summary`, `.order-summary-row`, `.cart-empty` CSS classes to `src/input.css` and built `css/style.css`.

Plan 06-02 added `loadCartFromSupabase` — called by `components.js initAuth` on `INITIAL_SESSION`, so authenticated users see their Supabase cart on page load without additional wiring in this file.

The cart store exposes: `items` (array), `count`, `total`, `totalFormatted`, `add`, `remove`, `setQty`, `clear`. Price formatting is `'Rp ' + value.toLocaleString('id-ID')`.

The `<main>` element uses `x-data` with an empty object (`{}` or bare `x-data`) — it reads exclusively from `$store.cart`. Do NOT create a local Alpine component with duplicate items state.
