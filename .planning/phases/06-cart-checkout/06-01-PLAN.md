# Plan 06-01: Phase 6 CSS — Cart Drawer, Cart Page, Checkout Page

## Goal
All CSS classes required by the mini cart drawer, full cart page, and 4-step checkout page exist in `src/input.css` and are compiled into `css/style.css`, so Plans 06-03 through 06-05 can reference classes without inventing inline styles.

## Context
`src/input.css` is the Tailwind v4 source file compiled via the Tailwind CLI into `css/style.css`. All existing component styles live inside an `@layer components { }` block starting at line 39. The file ends at line 1069 after the Phase 4 auth/account responsive block. Design tokens are defined in both `@theme` (Tailwind utilities) and `:root` (CSS custom properties): `--sage`, `--rose`, `--beige`, `--charcoal`, `--sage-light`, `--rose-light`, `--text-muted`, `--white`, `--lavender`.

The existing responsive block `@media (max-width: 900px)` (line 819) and `@media (max-width: 1023px)` (line 1045) must not be altered — new Phase 6 responsive rules are appended after the closing brace of the last existing media query.
