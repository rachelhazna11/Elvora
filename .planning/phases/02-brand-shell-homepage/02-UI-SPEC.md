## Design System

| Property | Value |

## Spacing Scale

Declared values (8-point base, multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon dot gaps, cart badge offset, scrollbar/border hairlines |
| sm | 8px | Marquee dot spacing, intra-component gaps (swatch row, star gap) |
| md | 16px | Default component padding (product-badge, wishlist-btn offset) |
| lg | 24px | Card inner padding (product-info), grid gaps (product grid), testimonial card padding |
| xl | 32px | Hero scroll indicator, modal close offset, footer bottom padding |
| 2xl | 48px | Section header bottom margin, hero actions margin-top, newsletter form max-width context |
| 3xl | 64px | Section horizontal padding (desktop), hero content left padding, footer top padding |
| 4xl | 80px | Hero top padding (below fixed nav), brand story section vertical, nav height offset |
| 5xl | 100px | Section vertical padding (default for all homepage sections) |

**Exceptions:**
- Nav horizontal padding: 48px (3xl) on desktop; collapses to 16px (md) on mobile
- Marquee section vertical padding: 16px (md) — grid-aligned, visually near-identical to prototype
- Touch targets (nav icons, wishlist-btn, cart badge, social-btn): minimum 44×44px per NF-001
- Newsletter section vertical padding: 80px (4xl) — compact by design
- Footer: 72px top / 40px bottom — matches prototype

## Typography

Four tiers declared. All sizes use Tailwind v4 utility classes (e.g., `text-[11px]` for non-standard).

| Tier | Family | Size | Weight | Line Height | Letter Spacing | Usage |
|------|--------|------|--------|-------------|----------------|-------|
| Display | Playfair Display | clamp(48px, 5vw, 72px) | 700 | 1.1 | — | Hero `h1`, homepage display headlines |
| Heading | Playfair Display | clamp(22px, 4vw, 52px) | 700 | 1.15–1.2 | — | Section titles (`section-title`), brand story title, product card names, collection names, modal titles, testimonial quote marks |
| Body | Poppins | 14–15px | 400 | 1.8 | — | Hero description, story text, testimonial body, footer tagline, nav links, CTA buttons, product badges, view-all links, section-labels, footer links |
| Caption | Poppins | 11–12px | 400 | 1.4 | 2–5px | Section labels (`section-label`), category counts, marquee items, cart badge, footer col titles, scroll indicator, hero-badge-text, stat-label |

**Active font weights — Poppins only:**
- 400 (regular) — body copy, captions, footer links
- 600 (semibold) — labels, button text, nav links, emphasis, price display

**Playfair Display weights used:** 700 only for Display and Heading tiers.

**Rules:**
- Display and Heading tiers: `font-family: var(--font-display)` (Playfair Display)
- All other tiers: `font-family: var(--font-body)` (Poppins)
- Italic emphasis (`em`) in headings: color `--color-rose` — used for emotional word stress ("*Wear Your*", "*Story*")
- No `font-size` below 16px on form inputs (prevents iOS Safari auto-zoom per NF-001)
- Newsletter input: use `font-size: 16px` on the actual `<input>` element to prevent iOS zoom
- Poppins weights 300 and 500 are NOT loaded and NOT used — remove from Google Fonts request

| Rose light | `--color-rose-light` | `#F5E8E8` | Product card bg tint 2 |

**60/30/10 mapping:**
- 60% dominant: beige `#F5F1ED` — the warm cream that backgrounds the entire browsing experience
- 30% secondary: charcoal `#2E2E2E` (text, brand story, footer, marquee) + white `#FFFFFF` (card surfaces)
- 10% accent: sage `#A8BFA3` as the primary accent; rose `#D8A7A7` as the secondary emotional accent

**Accent reserved specifically for:**
- `--color-sage`: section-label text, newsletter section background, category card 1, progress indicators, focus border, "New" product badge, Alpine selected states
- `--color-rose`: all hover states on interactive elements (nav links, CTAs, footer links, social buttons), italic emphasis in display headings, Sale badge, testimonial large quotation mark

**No default Tailwind framework colours (blue-500, etc.) may appear anywhere in the UI.**

## Interaction Contracts

### Nav Scroll Transparency (D-05)

- **Trigger:** `window.scroll` event inside Alpine `x-init`
- **State change:** `scrolled` boolean in Alpine component data
- **At rest (scrollY < 80):** `class="bg-transparent"` — nav is invisible over hero image
- **Scrolled (scrollY ≥ 80):** `class="bg-beige/90 backdrop-blur-md border-b border-sage/20"` — frosted glass
- **Transition:** CSS `transition: all 0.3s ease` on nav element
- **Non-index pages:** nav always uses frosted glass state (no transparent mode)

### Mobile Nav Drawer (D-04)

- **Trigger:** hamburger button click (visible below `md:` breakpoint only)
- **Alpine data:** `{ navOpen: false }` in `x-data` on nav root
- **Open:** `x-show="navOpen"` with `x-transition:enter="transition ease-out duration-300" x-transition:enter-start="translate-x-full" x-transition:enter-end="translate-x-0"` and reverse for leave
- **Drawer:** full viewport height, right-side slide-in, charcoal bg, contains all nav links (stacked, 16px, uppercase) + icons + close button
- **Close triggers:** close button, clicking backdrop overlay, Escape key

### FAQ Accordion (D-09)

- **Alpine data:** `{ activeIndex: null }` on the FAQ container
- **Item toggle:** `@click="activeIndex = activeIndex === i ? null : i"` per item button
- **Answer visibility:** `x-show="activeIndex === i"` with height animation via `x-transition`
- **Chevron rotation:** `:class="{ 'rotate-180': activeIndex === i }"` on SVG icon with `transition: transform 0.2s`

### Newsletter Form Submit (D-10)

- **Alpine data:** `{ email: '', submitted: false, error: false, loading: false }`
- **Validation:** HTML5 `type="email"` + Alpine `x-on:submit.prevent` checks non-empty before calling Supabase
- **Loading state:** submit button shows "…" text, disabled, cursor-not-allowed
- **Success:** `submitted = true` → swap form HTML for "You're on the list." message (no page reload)
- **Duplicate email:** Supabase will return a unique constraint error — display "You're already subscribed." in success style
- **Network error:** `error = true` → show "Something went wrong — please try again." below form in `rgba(255,255,255,0.8)` text

### Best Sellers Data Fetch (D-11)

- **Alpine data:** `{ products: [], loading: true }` with `async init()` lifecycle hook
- **Query:** `supabase.from('products').select('id, name, price, product_images(*), product_variants(color_name, color_hex)').order('created_at', { ascending: false }).limit(4)`
- **Skeleton state:** while `loading === true`, render 4 placeholder cards with `animate-pulse bg-sage-light`
- **Render:** `x-for="product in products"` with `template` tag
- **Error:** if fetch fails, render 4 hardcoded static fallback cards (no broken layout)

### Testimonials Data Fetch (D-11)

- **Alpine data:** `{ testimonials: [], loading: true }` with `async init()`
- **Query:** `supabase.from('testimonials').select('*').eq('is_active', true).limit(3)`
- **Skeleton:** 3 beige pulsing card placeholders while loading
- **Render:** `x-for="t in testimonials"`

## Performance Contracts

Per NF-002:

- Hero image: `fetchpriority="high"` attribute on `<img>`
- All below-fold images: `loading="lazy"` attribute
- All `<img>` tags: explicit `width` and `height` attributes to prevent CLS
- No render-blocking scripts in `<head>` — Alpine.js loaded with `defer`, Supabase SDK loaded as ES module via `<script type="module">`
- Tailwind CSS: compiled via CLI to `css/style.css` — Play CDN never used in production
- Google Fonts: `&display=swap` parameter on all font requests
- Supabase image URLs: append `?width=800&quality=80` to all product images, `?width=400&quality=75` for card thumbnails

---

## Accessibility Contracts

Per NF-003 (WCAG 2.1 AA):

- All images: meaningful `alt` attributes; decorative images (lookbook grid): `alt=""`
- Colour contrast: charcoal `#2E2E2E` on beige `#F5F1ED` = ~10:1 ratio (passes AA and AAA)
- Colour contrast: white text on sage `#A8BFA3` = ~3.5:1 (passes AA for large text — use for newsletter section headings only)
- All interactive elements keyboard-reachable and operable
- Focus styles: default browser outline retained; not removed without replacement
- Mobile nav drawer: focus trap when open; Escape closes
- Hamburger button: `aria-label="Open navigation menu"` / `aria-label="Close navigation menu"` toggle
- FAQ accordion: `aria-expanded` on trigger buttons
- Nav icon buttons: `aria-label="Search"` on search icon, `aria-label="Your account"` on user icon, `aria-label="Shopping bag"` on bag icon
- Form inputs: `<label>` elements associated via `for`/`id` (newsletter input has implicit label via placeholder — add `aria-label="Email address"`)
- Heading hierarchy: `h1` on hero headline only; `h2` on all section titles; `h3` on card/item names
