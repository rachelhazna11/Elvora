## Interaction Contracts

### Auth Page — Tab Toggle

- Alpine `x-show` switches between sign-in and sign-up form panels. No page reload.
- `?tab=signup` URL param: on `init()`, read `new URLSearchParams(location.search).get('tab')` to set active tab.
- Tab switch: 150ms opacity transition (`x-transition:enter="transition-opacity duration-150"`).
- Active tab has bottom border `2px solid var(--charcoal)`. Inactive tab: `color: var(--text-muted)`.

### Auth Page — Form Submission

- Sign In submit: show loading state on button ("Signing In…" text, `disabled` attribute). On success: redirect to `account.html`. On error: display error message inline below the form (not toast) — "Incorrect email or password. Try again."
- Sign Up submit: show loading state. On success: show Toastify toast "Welcome to Elvora, [first_name]!" (2s duration, green background matching `--sage`), then redirect to `account.html`. On error: display inline error.
- Password field: minimum 8 chars client-validated before submit. Confirm-password must match — validate on blur of confirm field.
- "Forgot password" link: navigates to Supabase hosted password reset flow (external) — no custom page needed for Phase 4.

### Account Dashboard — Tab Navigation

- Alpine `x-data` controls active tab. Three tabs: `'profile'`, `'wishlist'`, `'preferences'`.
- On `init()`: read `location.hash` (`#wishlist`, `#preferences`, defaults to `#profile`) to set initial active tab.
- Tab switch: no animation (instant panel swap via `x-show`). Hash updates on click: `location.hash = tab`.
- Sidebar item active state: `border-left: 2px solid var(--rose)` + `background: var(--rose-light)`.

### Wishlist Heart Toggle (PLP + PDP)

- **Guest state:** On heart click → Toastify toast ("Sign in to save items to your wishlist", 3s duration, charcoal background) + after 2s redirect to `auth.html?tab=signin`. Toast is clickable — clicking toast navigates immediately.
- **Logged-in optimistic toggle:** Heart icon state changes instantly (Alpine reactive `wishlisted` boolean). Supabase insert/delete fires async. On Supabase error: revert icon state + show error toast ("Couldn't save item — try again", rose background).
- **Filled heart visual:** `.wishlist-btn.active` — background `var(--rose)`, color white. Heart emoji: `♥` (filled) vs `♡` (outlined). Or use inline SVG for crisp rendering.
- **Initial state on load (logged in):** Single query on page init fetches all `wishlist_items.product_id` for user → stored as `Set` in Alpine data → each card reads `wishlistedIds.has(product.id)`.

### Profile Save

- Inline save — no modal or confirmation. "Save Changes" button. On submit: show loading state → success toast "Changes saved" (`--sage` background, 2s). On error: error toast "Couldn't save — try again" (`--rose` background).
- Email field: `readonly` attribute. Poppins 14px, `background: rgba(46,46,46,0.04)`, cursor: not-allowed.

### Style Preferences — Chip Select

- Single-select per field (activity, fit, aesthetic, colour). Clicking a selected chip deselects it (allows clearing a preference).
- Alpine `x-bind:class="{ 'selected': prefs.activity === 'padel' }"` pattern per chip.
- "Save Preferences" button: same loading + success/error toast pattern as Profile Save.

### Wishlist Remove (Account Dashboard)

- Remove button (×) on wishlist card: **optimistic** — card fades out (150ms opacity to 0, then removed from DOM via Alpine `x-for` reactive array). Supabase delete fires async. On error: card reappears + error toast.
- No confirmation modal needed (not destructive enough to warrant; item can be re-added from PLP/PDP).

## Accessibility Requirements (Phase 4 specific)

- All form inputs have associated `<label for="...">` elements — no `placeholder` as the only label
- Auth tab toggle: use `role="tab"`, `aria-selected`, `aria-controls` on tab buttons; `role="tabpanel"` on form containers
- Wishlist heart button: `aria-label="Add to wishlist"` / `aria-label="Remove from wishlist"` (dynamic based on state); `aria-pressed` reflects current state
- Sidebar navigation: `role="navigation"` wrapper, `aria-label="Account navigation"`, `aria-current="page"` on active item
- Modal/overlay: not needed for this phase (no modal dialogs introduced beyond existing size guide from Phase 3)
- Focus visible: all interactive elements must have visible focus ring — use `outline: 2px solid var(--sage); outline-offset: 2px` replacing any default browser outline removal
- Dropdown menu: `aria-expanded` on trigger; `role="menu"` on dropdown; `role="menuitem"` on items; Escape key closes
- Minimum touch target: all clickable elements 44×44px — apply `min-width: 44px; min-height: 44px` to smaller elements (heart button is 36px visual but padded to 44px touch area)
- Error messages: `role="alert"` on error containers so screen readers announce them immediately
- No form input with `font-size` below 16px on mobile (prevents iOS Safari zoom) — apply `font-size: 16px` on `<input>` within `@media (max-width: 767px)`
