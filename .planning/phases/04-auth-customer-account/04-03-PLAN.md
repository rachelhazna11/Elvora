    {
      tab: 'signin',
      loading: false,
      error: '',
      siEmail: '', siPassword: '',
      suFirst: '', suLast: '', suEmail: '', suPassword: '', suConfirm: '',

      async init() {
        const tab = new URLSearchParams(location.search).get('tab');
        if (tab === 'signup') this.tab = 'signup';
        const { data: { user } } = await window.supabase.auth.getUser();
        if (user) window.location.href = '/account.html';
      },

      async handleSignIn() {
        this.loading = true; this.error = '';
        try {
          await window.signIn(this.siEmail, this.siPassword);
          window.location.href = '/account.html';
        } catch (e) {
          this.error = 'Incorrect email or password. Please try again.';
        } finally { this.loading = false; }
      },

      async handleSignUp() {
        this.error = '';
        if (this.suPassword.length < 8) { this.error = 'Password must be at least 8 characters.'; return; }
        if (this.suPassword !== this.suConfirm) { this.error = 'Passwords do not match.'; return; }
        this.loading = true;
        try {
          await window.signUp(this.suEmail, this.suPassword, this.suFirst, this.suLast);
          Toastify({ text: 'Welcome to Elvora, ' + this.suFirst + '!', duration: 2000, gravity: 'bottom', position: 'right',
            style: { background: 'var(--sage)', color: '#fff', borderRadius: '50px', fontSize: '12px', letterSpacing: '2px' }
          }).showToast();
          setTimeout(() => { window.location.href = '/account.html'; }, 2000);
        } catch (e) {
          this.error = e.message?.includes('already registered') || e.message?.includes('already exists')
            ? 'An account with this email already exists.'
            : (e.message || 'Something went wrong. Please try again.');
        } finally { this.loading = false; }
      }
    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User input → Alpine x-data | Email, password, name fields bound via x-model |
| Alpine x-data → window.signIn/window.signUp | Credentials passed to auth service |
| Supabase Auth API → Browser | Error messages from Supabase returned to JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03-01 | Information Disclosure | Sign-in error message | mitigate | Map all signIn errors to generic "Incorrect email or password. Please try again." — never expose "email not found" vs "wrong password" distinction |
| T-04-03-02 | Spoofing | Client-side logged-in redirect in init() | accept | RLS is the real access gate; redirect is UX convenience only; bypassing it yields no data |
| T-04-03-03 | Tampering | x-text for error messages | mitigate | Alpine x-text escapes HTML; error string from Supabase is string not HTML; safe |
| T-04-03-04 | Information Disclosure | Supabase "already registered" error message | accept | Standard auth UX practice; user submitting their own email gets confirmation it exists — acceptable per ASVS L1 |
| T-04-03-05 | Tampering | Password min-length client validation | mitigate | 8-char minimum validated client-side before submit; Supabase Auth also enforces server-side password policy |
| T-04-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; CDN links already in project |
</threat_model>
