# Plan 07-02: Migration RLS + Best Sellers Fix

## Goal

Dua database gap yang memblokir fitur admin: (1) admin tidak bisa membaca orders karena policy SELECT hilang, (2) homepage menampilkan 4 produk terbaru bukan best sellers sebenarnya.

## Context

Audit RLS di `001_schema.sql` mengkonfirmasi dua policy yang HILANG:
- `orders` table: ada `orders_admin_update` dan `orders_admin_delete` tapi TIDAK ADA `orders_admin_select`. Tanpa ini, `supabase.from('orders').select(...)` mengembalikan array kosong untuk admin (RLS memfilter silently, tanpa error).
- `products` table: tidak ada `products_admin_select`. Admin hanya melihat produk `is_active=true` karena `products_public_read` memfilter. Draft produk tidak terlihat.

`index.html` best sellers section (baris 141-147) saat ini query: `.from('products').select(...).order('created_at', { ascending: false }).limit(4)` — tidak ada filter `is_best_seller`. Akibatnya, toggle admin tidak berpengaruh di homepage.

Ini adalah wave 1 plan yang bisa berjalan paralel dengan 07-01 karena menyentuh file yang berbeda sama sekali.

## Tasks

### Task 1: Buat Migration 007_admin_policies.sql

**File yang dibuat:** `supabase/migrations/007_admin_policies.sql`

Buat file migration baru dengan konten berikut. File ini diapply MANUAL ke Supabase melalui SQL Editor — bukan via Supabase CLI (sesuai pola proyek; lihat STATE.md "D-08 cloud-only constraint").

```sql
-- =============================================================================
-- Elvora — Admin RLS Policy Additions
-- File: supabase/migrations/007_admin_policies.sql
-- Phase 7: Admin Panel
-- Apply via: Supabase SQL Editor (Dashboard → SQL Editor → Run)
--
-- Adds missing admin SELECT policies identified during Phase 7 RLS audit:
--   1. orders_admin_select — allows admin to read ALL orders
--   2. products_admin_select — allows admin to read ALL products including inactive drafts
--
-- Pre-requisite: 001_schema.sql must already be applied (is_admin() function must exist)
-- Idempotent: uses CREATE POLICY IF NOT EXISTS pattern (PostgreSQL 15+)
-- If Supabase version does not support IF NOT EXISTS, run the DROP + CREATE pattern below.
-- =============================================================================

-- Drop existing policies first to make this idempotent
-- (safe to run multiple times — does nothing if policy doesn't exist)
drop policy if exists "orders_admin_select" on orders;
drop policy if exists "products_admin_select" on products;

-- Policy 1: Admin can SELECT all orders (required for orders read view in admin panel)
-- Without this, RLS silently returns empty rows for admin user on orders table.
create policy "orders_admin_select"
  on orders
  for select
  using (is_admin());

-- Policy 2: Admin can SELECT all products including inactive drafts
-- Without this, admin only sees is_active=true products (public read policy filters)
-- This is needed to manage draft products and toggle is_active in admin panel.
create policy "products_admin_select"
  on products
  for select
  using (is_admin());
```

Sertakan juga komentar ini di atas file untuk instruksi apply:

```
-- HOW TO APPLY:
-- 1. Open Supabase Dashboard → SQL Editor
-- 2. Paste this entire file content
-- 3. Click "Run"
-- 4. Verify: go to Authentication → Policies → check orders and products tables
--    for new policies "orders_admin_select" and "products_admin_select"
```

### Task 2: Fix Best Sellers Query di index.html

**File yang dimodifikasi:** `index.html`

Lokasi query yang perlu diubah: section `<!-- ═══ Best Sellers (Supabase) ═══ -->`, di dalam `async init()` method, baris yang saat ini berbunyi:

```
.from('products')
.select('id, name, price, product_images(*), product_variants(color_name, color_hex)')
.order('created_at', { ascending: false })
.limit(4);
```

Ubah menjadi:

```
.from('products')
.select('id, name, slug, base_price, price, product_images(*), product_variants(colour, colour_hex)')
.eq('is_best_seller', true)
.eq('is_active', true)
.order('created_at', { ascending: false })
.limit(4);
```

Perubahan yang dilakukan:
1. Tambahkan `.eq('is_best_seller', true)` — hanya tampilkan produk yang ditandai best seller
2. Tambahkan `.eq('is_active', true)` — pastikan hanya produk aktif yang tampil
3. Tambahkan `slug` ke select untuk link navigasi yang benar
4. Tambahkan `base_price` ke select (schema menggunakan `base_price`, bukan `price`)
5. Koreksi nama kolom variant: `colour` dan `colour_hex` (bukan `color_name` dan `color_hex` — sesuai schema 001_schema.sql)

Di bagian render products di section yang sama, pastikan template link menggunakan `product.slug`:
```
:href="'/product.html?slug=' + product.slug"
```
(saat ini mungkin menggunakan `?id=`, sesuaikan jika perlu — cek baris template x-for di section best sellers)

Juga update referensi harga di template: gunakan `product.base_price || product.price` agar backward compatible.

Tidak ada perubahan lain di `index.html`.
