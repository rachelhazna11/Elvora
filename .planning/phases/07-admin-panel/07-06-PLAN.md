# Plan 07-06: Orders Read View

## Goal

Admin dapat melihat seluruh daftar order dari checkout, expand setiap order untuk melihat line items (nama produk, variant, qty, harga) — semua diambil langsung dari Supabase.

## Context

Plan 07-01 membuat Orders section di `admin.html` dengan struktur tabel yang lengkap (termasuk expand row untuk order items) menggunakan `toggleOrderItems(o.id)` dan state `selectedOrderId`, `orderItems`, `loadingItems`. Plan ini hanya perlu:
1. Mengisi implementasi `adminGetOrders()` dan `adminGetOrderItems()` di `js/admin.js`
2. Mengupdate `loadSection('orders')` di adminApp() dengan error handling yang proper

**RLS requirement:** `orders_admin_select` policy harus sudah diapply via migration 007 (plan 07-02). Tanpa itu, query mengembalikan array kosong silently. Plan ini menambahkan pengecekan count dan pesan informatif jika orders kosong.

`order_items` schema (dari 001_schema.sql): `id, order_id, product_id, variant_id, product_name (snapshot), variant_label (snapshot), unit_price (snapshot), quantity, created_at`. Kolom snapshot (`product_name`, `variant_label`, `unit_price`) digunakan langsung — tidak perlu JOIN ke products/variants.

Plan ini bisa berjalan paralel dengan 07-03, 07-04, 07-05 karena menyentuh Orders section (section berbeda di admin.html) dan fungsi berbeda di admin.js.

## Tasks

### Task 1: Implement adminGetOrders() dan adminGetOrderItems() di js/admin.js

**File yang dimodifikasi:** `js/admin.js`

Ganti placeholder `adminGetOrders()` dengan implementasi penuh:

```javascript
async function adminGetOrders(limit = 50) {
  // Requires orders_admin_select RLS policy from migration 007
  // Without it, this returns [] silently — check data.length and inform user
  const { data, error } = await window.supabase
    .from('orders')
    .select(`
      id,
      created_at,
      status,
      subtotal,
      guest_email,
      user_id,
      shipping_address
    `)
    .order('created_at', { ascending: false })
    .limit(limit);

  if (error) throw error;
  return data || [];
}
```

Ganti placeholder `adminGetOrderItems()` dengan implementasi penuh:

```javascript
async function adminGetOrderItems(orderId) {
  const { data, error } = await window.supabase
    .from('order_items')
    .select(`
      id,
      product_name,
      variant_label,
      unit_price,
      quantity
    `)
    .eq('order_id', orderId)
    .order('id', { ascending: true });

  if (error) throw error;
  return data || [];
}
```

Kedua fungsi ini menggunakan snapshot columns (`product_name`, `variant_label`, `unit_price`) yang disimpan saat checkout — data konsisten meskipun produk asli diedit atau dihapus kemudian.

### Task 2: Update loadSection('orders') dan toggleOrderItems() di adminApp() di js/admin.js

**File yang dimodifikasi:** `js/admin.js`

Update `loadSection()` dispatcher untuk case `'orders'`:

```javascript
case 'orders':
  this.ordersLoading = true;
  try {
    this.orders = await adminGetOrders();
    // Reset any open order detail
    this.selectedOrderId = null;
    this.orderItems = [];
  } catch (err) {
    console.error('[admin] Load orders error:', err);
    showToast('Gagal memuat orders: ' + err.message, 'error');
  } finally {
    this.ordersLoading = false;
  }
  break;
```

Update `toggleOrderItems(orderId)` method di adminApp() — method ini sudah didefinisikan di plan 07-01 sebagai stub. Ganti dengan implementasi yang lebih robust:

```javascript
async toggleOrderItems(orderId) {
  // Collapse jika sudah terbuka
  if (this.selectedOrderId === orderId) {
    this.selectedOrderId = null;
    this.orderItems = [];
    return;
  }

  this.selectedOrderId = orderId;
  this.orderItems = [];
  this.loadingItems = true;

  try {
    this.orderItems = await adminGetOrderItems(orderId);
  } catch (err) {
    console.error('[admin] Load order items error:', err);
    showToast('Gagal memuat order items: ' + err.message, 'error');
    this.selectedOrderId = null;
  } finally {
    this.loadingItems = false;
  }
},
```

### Task 3: Perbaiki Orders Section UI di admin.html

**File yang dimodifikasi:** `admin.html`

Plan 07-01 sudah membuat struktur tabel orders yang lengkap. Namun ada satu hal yang perlu diperbaiki: `x-show` pada expand row untuk order items.

Temukan baris di orders section:
```html
<tr x-show="selectedOrderId === o.id">
```

Pastikan baris ini juga memiliki `style="display:none;"` untuk mencegah flash saat Alpine belum init:
```html
<tr x-show="selectedOrderId === o.id" style="display:none;">
```

Juga tambahkan empty state yang informatif jika orders kosong karena migration belum diapply:
Temukan `<div class="admin-empty-state" x-show="!ordersLoading && orders.length === 0">No orders yet.</div>`
Ubah menjadi:
```html
<div class="admin-empty-state" x-show="!ordersLoading && orders.length === 0">
  <p style="margin-bottom:8px;">Belum ada orders.</p>
  <p style="font-size:12px;color:var(--text-muted);">Jika sudah ada checkout, pastikan migration <code>007_admin_policies.sql</code> sudah diapply ke Supabase.</p>
</div>
```

Tambahkan juga format order ref yang lebih readable. Di dalam template x-for orders, baris:
```html
<td x-text="o.id.substring(0,8).toUpperCase()"></td>
```
Ubah menjadi:
```html
<td>
  <span style="font-family:monospace;font-size:12px;letter-spacing:1px;" x-text="o.id.substring(0,8).toUpperCase()"></span>
</td>
```

Tambahkan subtotal yang diformat dengan rupiah jika belum ada format yang benar. Verifikasi baris:
```html
<td x-text="'Rp ' + Number(o.subtotal).toLocaleString('id-ID')"></td>
```
Sudah benar — tidak perlu diubah.

Untuk tanggal order, pastikan format yang readable:
```html
<td x-text="new Date(o.created_at).toLocaleDateString('id-ID', {day:'numeric', month:'short', year:'numeric'})"></td>
```

Untuk customer info, tampilkan email atau indicator "Registered User":
```html
<td x-text="o.guest_email || '—'"></td>
```
