# Plan 07-03: Products List + Delete

## Goal

Admin dapat melihat seluruh katalog produk (termasuk draft/inactive) dalam tabel yang bisa di-scroll, dan menghapus produk beserta semua data terkait (variants, images) dengan konfirmasi.

## Context

Plan 07-01 telah membuat shell `admin.html` dengan section products yang memiliki `products: []` dan tombol Delete yang memanggil `deleteProduct(p.id, p.name)`. Plan 07-02 telah menambahkan `products_admin_select` RLS policy sehingga query tanpa filter `is_active` berhasil.

`js/admin.js` saat ini masih berisi placeholder functions yang mengembalikan `[]`. Plan ini mengisi implementasi penuh `adminGetProducts()` dan `adminDeleteProduct()`, serta method `deleteProduct()` di Alpine `adminApp()`.

Setelah plan ini, admin bisa melihat dan menghapus produk. Plan 07-04 akan menambahkan form create/edit.

## Tasks

### Task 1: Implement adminGetProducts() di js/admin.js

**File yang dimodifikasi:** `js/admin.js`

Ganti fungsi placeholder `adminGetProducts()` yang mengembalikan `[]` dengan implementasi penuh:

```javascript
async function adminGetProducts() {
  // Requires products_admin_select RLS policy from migration 007
  // Returns ALL products including is_active=false (drafts)
  const { data, error } = await window.supabase
    .from('products')
    .select(`
      id,
      name,
      slug,
      base_price,
      is_active,
      is_best_seller,
      category_id,
      description,
      fabric_details,
      care_instructions,
      created_at,
      categories(id, name),
      product_images(id, url, alt_text, display_order),
      product_variants(id, colour, colour_hex, size, stock_quantity, sku)
    `)
    .order('created_at', { ascending: false });
  if (error) throw error;
  return data || [];
}
```

Catatan penting:
- Tidak ada filter `is_active` — admin melihat semua produk
- Sertakan relasi `categories`, `product_images`, dan `product_variants` agar form edit bisa pre-populate data
- Kolom yang digunakan sesuai schema `001_schema.sql`: `base_price` (bukan `price`), `colour`/`colour_hex` (bukan `color_name`/`color_hex`)

### Task 2: Implement adminDeleteProduct() di js/admin.js

**File yang dimodifikasi:** `js/admin.js`

Ganti fungsi placeholder `adminDeleteProduct(id)` dengan implementasi penuh:

```javascript
async function adminDeleteProduct(id) {
  // Step 1: Fetch product_images to get Storage paths for cleanup
  // We need the URL to derive the storage path for deletion
  const { data: images, error: imgFetchErr } = await window.supabase
    .from('product_images')
    .select('id, url')
    .eq('product_id', id);
  if (imgFetchErr) throw imgFetchErr;

  // Step 2: Delete from Storage (product-images bucket)
  // URL format: https://{project}.supabase.co/storage/v1/object/public/product-images/products/{id}/...
  // Extract path after "product-images/" for storage.remove()
  if (images && images.length > 0) {
    const storagePaths = images
      .map(img => {
        // Extract path from URL: everything after "/product-images/"
        const match = img.url.match(/\/product-images\/(.+)$/);
        return match ? match[1] : null;
      })
      .filter(Boolean);

    if (storagePaths.length > 0) {
      // Storage delete errors are non-blocking — log but continue
      const { error: storageErr } = await window.supabase.storage
        .from('product-images')
        .remove(storagePaths);
      if (storageErr) {
        console.warn('[admin] Storage cleanup error (non-blocking):', storageErr.message);
      }
    }
  }

  // Step 3: Delete product_images rows from DB
  // (cascade delete via FK may handle this, but explicit is safer)
  const { error: imgDelErr } = await window.supabase
    .from('product_images')
    .delete()
    .eq('product_id', id);
  if (imgDelErr) throw imgDelErr;

  // Step 4: Delete product_variants rows
  const { error: varDelErr } = await window.supabase
    .from('product_variants')
    .delete()
    .eq('product_id', id);
  if (varDelErr) throw varDelErr;

  // Step 5: Delete the product itself
  const { error: prodDelErr } = await window.supabase
    .from('products')
    .delete()
    .eq('id', id);
  if (prodDelErr) throw prodDelErr;
}
```

Catatan:
- Step 1-3 (Storage + product_images) dilakukan sebelum delete product agar FK constraint tidak diblokir
- Storage error bersifat non-blocking (log warning) — data di DB lebih penting dari file orphan
- Urutan delete: images → variants → product (urutan terbalik dari create)

### Task 3: Tambah deleteProduct() method ke adminApp() di js/admin.js

**File yang dimodifikasi:** `js/admin.js`

Di dalam `adminApp()` object, tambahkan method `deleteProduct(id, name)`:

```javascript
async deleteProduct(id, name) {
  const confirmed = window.confirm(
    `Hapus produk "${name}"?\n\nTindakan ini akan menghapus produk, semua varian, dan gambar secara permanen.`
  );
  if (!confirmed) return;

  try {
    await adminDeleteProduct(id);
    // Refresh product list after successful delete
    this.products = await adminGetProducts();
    showToast(`Produk "${name}" berhasil dihapus`);
  } catch (err) {
    console.error('[admin] Delete product error:', err);
    showToast(`Gagal menghapus produk: ${err.message}`, 'error');
  }
},
```

Pastikan method ini ditambahkan di dalam object yang dikembalikan oleh `adminApp()`, bersama method lain seperti `init()`, `setSection()`, `loadSection()`, dll.

Juga update `loadSection('products')` di adminApp():
```javascript
case 'products':
  this.productsLoading = true;
  try {
    this.products = await adminGetProducts();
  } catch (err) {
    showToast('Gagal memuat produk: ' + err.message, 'error');
  } finally {
    this.productsLoading = false;
  }
  break;
```

Gunakan try/catch/finally pattern yang sama untuk semua loadSection cases — ini lebih robust dari versi plan 07-01 yang tidak memiliki error handling.
