## Visuals

### Primary Screen Focal Points

- **PLP primary anchor:** Product card image — the image-first card layout draws the eye before name or price. Cards must allocate at minimum 60% of card height to the product image; name and price sit in a compact info strip below.
- **PDP primary anchor:** Main Swiper gallery image — full-bleed left column on desktop (55% width), `fetchpriority="high"` on first slide `<img>`. The gallery is the single most prominent element on the page and must load fast and render at full quality before all other PDP sections.

### Visual Hierarchy Rules

- PLP: image → price → name → swatches (reading order matches visual weight)
- PDP: gallery (left) → product name + price (top right) → size/colour selectors → ATC button — all right-column elements stack in this order on mobile
- Section headings ("Complete the Look", "You Might Also Love", "What Our Community Says") use `.section-label` above Playfair Display heading — same double-header pattern as Phase 2 homepage sections
- White product cards (`--white`) float above the `--beige` page background — no additional border or shadow needed on default state; `box-shadow` only on hover

### Image Treatment

- All product card images: aspect-ratio 3/4 (portrait), `object-fit: cover`, `object-position: top`
- PDP main gallery: aspect-ratio 4/5 on mobile, 3/4 on desktop
- Complete the Look cards: narrower variant, aspect-ratio 3/4

## Interaction Contracts

### PLP Filter + Sort
- Activity tab click → fires Supabase re-fetch → updates `products` array in Alpine x-data → grid re-renders via `x-for`
- Sub-category pill row appears below activity tabs only when a non-"All" activity is selected (Alpine `x-show`)
- Active pill: `background: var(--charcoal); color: white; border-color: var(--charcoal)`
- URL sync: on filter/sort change → `history.replaceState` with `?category=padel&sub=matcha-babe&sort=price-asc` — no page reload
- On page load: Alpine `init()` reads `URLSearchParams` and initialises filter state before first fetch
- Colour swatch hover on card: Alpine `x-bind:style` updates card `background-image` to variant image URL (pre-loaded in products array)

### Nav Search (click-to-expand)
- Search icon click → Alpine `x-show` + CSS transition (`opacity 0→1, width 0→240px`) expands input in-place in nav right side
- Click outside or Escape → collapses input, clears query, hides dropdown
- Min 2 characters → debounce 300ms → Supabase ilike query → show dropdown
- Dropdown max 6 results; "No results" row shown if empty
- Suggestion click → navigate to `product.html?slug={slug}`
- Enter key → navigate to `shop.html?search={query}`
- Dropdown z-index: 200 (above nav content, below modal overlay)

### PDP Gallery (Swiper.js)
- Swiper configured with `thumbs` module: main image slider + thumbnail slider
- Main image: loop enabled, lazy loading for images beyond first
- Thumbnail click → updates main image (Swiper `thumbs.swiper` sync)
- Swipe gesture on mobile: enabled by default in Swiper
- Image URLs use Unsplash format: `?w=1200&q=80` for main, `?w=200&q=60` for thumbnails (D-11)

### Colour Swatch → Image Update (PDP)
- Swatch click → updates Alpine `selectedColour` → updates main Swiper slide index to first image of that variant
- Active swatch: 2px solid ring, 3px offset
- Sticky ATC bar reflects `selectedColour` reactively via Alpine

### Size Selector (PDP)
- Size click → updates Alpine `selectedSize`
- No size selected: ATC button shows "Select a Size" (disabled, opacity 0.7)
- Size selected: ATC button shows "Add to Bag" (enabled, `.btn-primary` style)
- Sticky ATC bar shows selected size name reactively

### Size Guide Modal (F-015)
- "Size Guide" link click → `showSizeGuide = true` (Alpine)
- Modal renders via `x-show` with `x-transition` (fade + scale from 0.95)
- Focus trap: first focusable element receives focus on open; Tab cycles within modal; Shift+Tab reverses
- Close via: × button, Escape key (`@keydown.escape.window="showSizeGuide = false"`), click on overlay backdrop
- Modal content: static HTML table (not Supabase) — XS/S/M/L/XL × Bust/Waist/Hips in cm + brief measure instructions

### Sticky ATC Bar (F-014)
- IntersectionObserver watches the main `.atc-main` button
- When `.atc-main` exits viewport (scrolled past): Alpine `stickyVisible = true` → `.sticky-atc` appears via `x-show` with slide-up transition
- When `.atc-main` re-enters viewport: `stickyVisible = false` → sticky bar hides
- Sticky bar content: product name (truncated to 1 line), selected colour + size (or "—" if unselected), price, "Add to Bag" button
- Clicking sticky ATC → same `Alpine.store('cart').add()` call as main button

### Add to Cart (Stub)
- Calls `Alpine.store('cart').add({ productId, variantId, quantity: 1 })` — store method stubbed in Phase 2, actual logic Phase 6
- For Phase 3: the call fires but cart state changes are cosmetic/stub only
- On successful call (no error thrown): show Toastify toast "Added to Bag" — charcoal bg, white text, 2.5s duration, bottom-right position

### Reviews Load More (F-019)
- Initial load: 5 reviews from Supabase (`.limit(5)`)
- "Load More" click: fetch next 5 with `.range(offset, offset+4)`, append to reviews array
- Hide "Load More" button when all reviews loaded (compare loaded count to aggregate total)

### Accordion (Fabric & Care)
- Click accordion trigger → toggles `open` boolean in Alpine x-data per item
- Body reveals via `x-show` with CSS `max-height` transition (smooth expand)
- Icon: "+" when closed, "−" when open

## Accessibility Contract

All requirements are mandatory per REQUIREMENTS.md §NF-003 (WCAG 2.1 AA):

- Filter pills and size buttons: `role="button"` or native `<button>`. Keyboard: Space/Enter activates.
- Active filter pill: `aria-pressed="true"` on the active button.
- Colour swatches: `<button aria-label="{colour name}">`. Active: `aria-pressed="true"`.
- Size buttons: `<button aria-label="Size {size}" aria-disabled="true">` for sold-out.
- Size guide modal: `role="dialog"`, `aria-modal="true"`, `aria-labelledby` pointing to modal title `<h2>`. Focus trap required (Tab / Shift+Tab cycles within modal). Escape closes.
- Sticky ATC bar: `aria-hidden="true"` when not visible (`x-bind:aria-hidden="!stickyVisible"`).
- Swiper gallery: main image `<img alt="{product name} - image {N} of {total}">`. Thumbnails `<button aria-label="View image {N}">`.
- Wishlist button: `aria-label="Save to wishlist"` (Phase 3: static, no aria-pressed toggle).
- Accordion: `<button aria-expanded="false/true" aria-controls="{body-id}">`. Body: `id="{body-id}" role="region"`.
- Review fit badge: text is sufficient — no additional aria needed.
- Form inputs (search): `<label for>` or `aria-label`. `autocomplete="off"`.
- Color contrast: `var(--charcoal)` (#2E2E2E) on `var(--beige)` (#F5F1ED) = ~12:1 — passes AA. `var(--text-muted)` (#8A8A8A) on white = ~3.5:1 — passes AA for large text / UI components; use sparingly for body text (borderline).
- No form input below 16px `font-size` on mobile (prevents iOS Safari auto-zoom per NF-001).

Source: REQUIREMENTS.md §NF-003, CONTEXT.md §Specific Ideas

## Performance Contract

Per REQUIREMENTS.md §NF-002:

- PLP initial load: fetch all active products in one Supabase query with joins (variants + images); avoid N+1 per card.
- Product card images: `loading="lazy"` for cards below fold. First 4 cards: `loading="eager"`.
- PDP main image: `fetchpriority="high"` on first Swiper slide image.
- PDP gallery: Swiper lazy loading for slides beyond the first (`lazy: true` in Swiper config).
- Image sizing:
  - PLP card: `?w=800&q=80` (Unsplash format per D-11)
  - PDP main: `?w=1200&q=80`
  - PDP thumbnail: `?w=200&q=60`
  - Complete the Look card: `?w=600&q=80`
  - Related products card: `?w=800&q=80`
- All `<img>` tags must have `width` and `height` attributes set to prevent CLS.
- Supabase queries: use `.select()` with explicit column lists — do not `select('*')` on joined tables.
