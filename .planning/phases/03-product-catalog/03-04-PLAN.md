## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL ?slug= param → Supabase query | User-controlled slug passed to getProduct() |
| Supabase product data → DOM | Product name/description/fabric rendered via Alpine x-text |
| Modal focus trap | Keyboard events intercepted within modal boundary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-14 | Tampering | ?slug= URL param passed to getProduct() | mitigate | getProduct() uses `.eq('slug', slug)` — Supabase SDK parameterized query; no SQL injection. Missing slug → redirect to shop.html (D-13) |
| T-03-15 | Tampering | XSS via product description, fabric_details, care_instructions | mitigate | All text-bearing fields use Alpine `x-text` (text node) — XSS-safe. Never use `x-html` for Supabase product data |
| T-03-16 | Spoofing | Swiper CDN integrity | accept | jsDelivr is a well-established public CDN; Swiper v11 is an established open-source library (MIT). No SRI hash required at this assessment scope. |
| T-03-17 | Denial of Service | Rapid Swiper init on malformed DOM | accept | Swiper is initialized inside `$nextTick` after Alpine renders the template — DOM is always ready. No malformed-DOM risk. |
| T-03-18 | Elevation of Privilege | Size guide modal focus escape | mitigate | Focus trap via `@keydown.window="trapFocus($event)"` — Tab/Shift+Tab cycles within modal; Escape closes modal and returns focus per Pitfall 5 pattern |
</threat_model>
