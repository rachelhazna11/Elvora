## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | product slug input to getProduct() | mitigate | `.eq('slug', slug)` is a parameterized Supabase SDK call — no raw SQL concatenation; immune to SQL injection |
| T-03-02 | Tampering | search query input to searchProducts() | mitigate | `.ilike('name', '%${query}%')` is parameterized — SDK handles escaping; 2-char minimum enforced |
| T-03-03 | Elevation of Privilege | product_pairings table write access | mitigate | Migration includes admin_write policy using `is_admin()`; public gets SELECT only via `USING (true)` policy |
| T-03-04 | Information Disclosure | RLS disabled on product_pairings | mitigate | Migration explicitly runs `ALTER TABLE product_pairings ENABLE ROW LEVEL SECURITY` before inserting policies |
| T-03-05 | Tampering | collection_id / product_id inputs to collection_products gap fill | accept | Seed data uses hardcoded UUIDs — no user input involved in the migration; conflict handled with ON CONFLICT DO NOTHING |
</threat_model>
