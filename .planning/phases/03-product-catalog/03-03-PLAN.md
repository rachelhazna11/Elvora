<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Search input → Supabase | User-typed query string sent as ilike parameter |
| Autosuggest result → DOM | Product name from Supabase rendered as x-text |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | Tampering | Search query text passed to ilike | mitigate | `window.searchProducts()` uses Supabase SDK `.ilike()` — parameterized; no SQL injection. 2-char minimum prevents excessive queries |
| T-03-11 | Tampering | XSS via product name in autosuggest | mitigate | Product names rendered via Alpine `x-text` (text node) — not innerHTML. XSS-safe |
| T-03-12 | Tampering | URL manipulation via goToSearch() | accept | Query is URL-encoded via `encodeURIComponent()`. PLP reads the value and passes it to ilike — parameterized query, no injection vector |
| T-03-13 | Denial of Service | Rapid keystroke search firing many requests | mitigate | 300ms debounce via clearTimeout/setTimeout pattern in onSearchInput() limits requests to at most 3/second |
</threat_model>
