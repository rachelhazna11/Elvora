Extend product.html in two steps:

STEP A — Add state and methods to the existing Alpine x-data object:

PREREQUISITE — Expose `supabase` on `window` in `js/supabase.js`:
Open `js/supabase.js` and add the following line at the very bottom of the file (after the `export const supabase = ...` declaration):
`window.supabase = supabase;`
This is required because `loadPairings()`, `loadRelated()`, and `loadReviews()` are defined inside Alpine inline `x-data` strings, which execute in the global scope and cannot access ES module exports directly. Adding `window.supabase` gives them access without breaking the module import pattern used by `js/products.js`. (Option B -- moving all methods to `js/products.js` and window-exposing them -- is also valid but inconsistent with the existing pattern where product.html Alpine x-data owns page-level state methods.)

Add these state properties (if not already present from Plan 04):
- `pairings: []`, `relatedProducts: []`, `reviews: []`, `reviewsOffset: 0`, `reviewsPageSize: 5`, `reviewsTotal: 0`, `allReviewsLoaded: false`, `reviewsLoading: false`, `averageRating: '0.0'`, `totalReviewCount: 0`

Note: `averageRating` is now a plain data property (not a computed getter) because it is populated by a dedicated aggregate query (see loadAggregateRating below). This ensures the displayed rating is always accurate regardless of how many review text rows have been paginated in.

Add these methods to the Alpine x-data object (called from within init() after `this.loading = false`):

`async loadAggregateRating()`: Fetch ALL ratings for this product without pagination to compute a stable average. Query: `window.supabase.from('reviews').select('rating').eq('product_id', this.product.id)`. From the returned array compute: `const ratings = (data || []).map(r => r.rating); this.totalReviewCount = ratings.length; this.averageRating = ratings.length ? (ratings.reduce((s, r) => s + r, 0) / ratings.length).toFixed(1) : '0.0';` This is separate from the paginated review text query so the aggregate is always based on all reviews, not just the loaded page. (Fixes Pitfall 7 from RESEARCH.md.)

`async loadPairings()`: Query `window.supabase.from('product_pairings').select('paired_product_id, display_order, products!product_pairings_paired_product_id_fkey(id, name, slug, base_price, product_images(url, alt_text, display_order, colour))').eq('product_id', this.product.id).order('display_order').limit(3)`. Map results to flat pairing card objects. Set `this.pairings`.

`async loadRelated()`: Query `window.supabase.from('products').select('id, name, slug, base_price, product_images(url, alt_text, display_order, colour), product_variants(colour, colour_hex)').eq('category_id', this.product.category_id).eq('is_active', true).neq('id', this.product.id).limit(4)`. Set `this.relatedProducts`.

`async loadReviews()`: Set `this.reviewsLoading = true`. Query: `window.supabase.from('reviews').select('id, reviewer_name, rating, review_text, fit_feedback, created_at', { count: 'exact' }).eq('product_id', this.product.id).order('created_at', { ascending: false }).range(this.reviewsOffset, this.reviewsOffset + this.reviewsPageSize - 1)`. Append data to `this.reviews`; increment offset by pageSize; if `count` is returned update `this.reviewsTotal` (note: totalReviewCount is already set by loadAggregateRating for display; reviewsTotal drives the load-more logic); set `this.allReviewsLoaded = this.reviews.length >= (this.reviewsTotal || this.totalReviewCount)`. Set `reviewsLoading = false`.

All three methods must use `window.supabase` (not bare `supabase`) because they execute in Alpine inline x-data context where ES module exports are not in scope. The `window.supabase = supabase` line added to `js/supabase.js` makes this work.

Update `init()` to call after `this.loading = false`:
`await Promise.all([this.loadAggregateRating(), this.loadPairings(), this.loadRelated(), this.loadReviews()]);`

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Supabase reviews data → DOM | Review text and reviewer names from Supabase rendered as text content |
| Supabase product_pairings → PDP cards | Pairing product names/images from Supabase rendered as text/src |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-19 | Tampering | XSS via review_text, reviewer_name | mitigate | All review data rendered via Alpine `x-text` — text node, not innerHTML. XSS-safe even if review text contains script tags |
| T-03-20 | Tampering | product_pairings joined product data | accept | product_pairings is admin-seeded data (RLS admin_write policy); public can only SELECT — no user-submitted pairing data |
| T-03-21 | Information Disclosure | RLS on reviews table | accept | reviews table has RLS from 001_schema.sql (public read, admin write per Phase 1 schema) — no private data exposed |
| T-03-22 | Tampering | Pairing card href injection via product slug | accept | Slug from Supabase products table (admin-controlled) rendered via Alpine :href binding — Alpine escapes attribute values; no user-supplied URL |
</threat_model>
