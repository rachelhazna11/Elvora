## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL query params → Alpine state | User-controlled URL params read by URLSearchParams on page load |
| Alpine x-text bindings → DOM | Product names/prices from Supabase rendered as text content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Tampering | URL ?category= param | accept | Value is used only as a lookup key in ACTIVITY_TOP_SLUGS — any unrecognised value returns empty slugs array, which falls through to unfiltered query. No injection possible via parameterized SDK calls. |
| T-03-07 | Tampering | XSS via product name | mitigate | All product name/price bindings use Alpine `x-text` (text node, not innerHTML) — XSS-safe. Never use `x-html` for Supabase data. |
| T-03-08 | Tampering | URL ?sort= param | accept | Sort value used as a switch key; unrecognised values fall through to default 'new-arrivals' sort. No SQL injection possible. |
| T-03-09 | Spoofing | Colour swatch image URL via variant data | accept | Images are Unsplash CDN URLs stored in product_images table; no user-supplied URL; RLS enforces public-read-only for products table. |
</threat_model>
