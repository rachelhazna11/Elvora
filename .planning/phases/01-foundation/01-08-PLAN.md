<objective>
Fix the major UAT gap from Test 1: Supabase client crashes on cold start with "Error: supabaseUrl is required" because SUPABASE_URL and SUPABASE_ANON_KEY are never set in the Netlify deployment environment.

Purpose: The Supabase JS SDK requires a valid URL and anon key at initialisation. Currently js/supabase.js reads from import.meta.env (undefined without Vite) and window.__ENV (never populated). The fix generates js/__env.js at Netlify build time using shell echo, injecting the Netlify environment variables into window.__ENV before any module runs.

Output: Updated netlify.toml (build command generates __env.js), updated .gitignore (excludes the generated file), updated .env.example (documents required env vars), and all 11 HTML files with the __env.js script tag inserted before supabase.js.
</objective>

    SUPABASE_ANON_KEY=your-anon-key-here

    Include a comment above each group making clear these are set in the Netlify dashboard under Site Settings > Environment Variables, not committed as a .env file. Also note that SUPABASE_ANON_KEY is a publishable key (safe to expose per Supabase RLS security model — it is not a secret key).

    Do not add SERVICE_ROLE_KEY or any other key to .env.example — only the two vars consumed by window.__ENV generation belong here.
