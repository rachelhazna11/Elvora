<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Anonymous → Supabase RLS | All public data access flows through RLS; anon key is safe to expose |
| Admin JWT → is_admin() | Admin privilege determined by app_metadata.role = 'admin' in JWT, not client-supplied data |
| User → own rows | user_id = auth.uid() predicates prevent cross-user data access |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | is_admin() function | mitigate | Use SECURITY DEFINER and read only from auth.jwt() -> 'app_metadata'; role cannot be set by user-editable raw_user_meta_data |
| T-01-02 | Information Disclosure | products table public SELECT | accept | Products are intended to be public; no PII in products table |
| T-01-03 | Information Disclosure | user_profiles RLS | mitigate | SELECT policy uses auth.uid() = id; users cannot read each other's profiles |
| T-01-04 | Information Disclosure | ai_style_sessions guest rows | mitigate | Guest sessions stored with null user_id; session_token used for retrieval; no PII linked |
| T-01-05 | Tampering | order_items unit_price | mitigate | unit_price is snapshotted at insert time (not joined to live products); no client-side price injection possible via RLS insert-only pattern |
| T-01-SC | Tampering | npm/pip/cargo installs | accept | This plan has no package installs — SQL file only |
</threat_model>
