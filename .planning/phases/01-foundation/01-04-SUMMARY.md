# Phase 1 Plan 4: Repository Scaffold Summary

**One-liner:** Full directory scaffold with 11 HTML shells, Supabase JS singleton, 5 stub modules, Tailwind v4 brand tokens, .env.example, and .gitignore.

## What Was Built

This plan created the complete structural backbone for the Elvora repository — 19 files that all downstream phases (2–7) build content into without restructuring the project.

### Task 1: HTML Page Shells + Tailwind CSS Source

All 11 HTML page shells created at the project root, each following the exact boilerplate from PATTERNS.md:
- `DOCTYPE html`, `<html lang="en">`, charset, viewport, page-specific title ("ELVORA — [Name]")
- Google Fonts link (Playfair Display italic/weight variants + Poppins weights 300–600)
- `/css/style.css` link (compiled output — gitignored; Netlify builds it)
- Single `<main><!-- Phase X fills this --></main>` body
- `<script type="module" src="/js/supabase.js"></script>` at end of body
- Zero inline JS, zero inline styles (NF-006 compliant)

`src/input.css` created with:
- `@import "tailwindcss"` at top
- `@theme` block: 8 colour tokens (#A8BFA3 sage, #D8A7A7 rose, #F5F1ED beige, #2E2E2E charcoal, #C6B7E2 lavender, #E8F0E6 sage-light, #F5E8E8 rose-light, #8A8A8A text-muted) + 2 font tokens (Playfair Display display, Poppins body)
- `:root` block mirroring tokens as native CSS custom properties for non-Tailwind CSS
- `* { box-sizing: border-box }`, `html { scroll-behavior: smooth }`, `body` base with Poppins, beige background, charcoal text, no overflow-x

### Task 2: JS Module Stubs + Config Files

`js/supabase.js` — shared client singleton:
- Imports `createClient` from Supabase CDN ESM URL (no npm needed)
- Reads `SUPABASE_URL` and `SUPABASE_ANON_KEY` via `import.meta.env?.VITE_*` with `window.__ENV` fallback (Netlify-compatible)
- Exports `const supabase` — the single instance all modules import
- Comment explicitly warns against service role key (T-04-03 mitigation)

Five stub modules (`js/auth.js`, `js/cart.js`, `js/products.js`, `js/style-match.js`, `js/admin.js`):
- Each imports `{ supabase }` from `./supabase.js`
- Each exports named async functions with `// TODO: Phase N` comments
- auth.js: signUp, signIn, signOut, getUser, onAuthChange (Phase 4)
- cart.js: getCart, addToCart, removeFromCart, updateQuantity, clearCart (Phase 6)
- products.js: getProducts, getProduct, getCategories, searchProducts (Phase 3)
- style-match.js: submitStyleMatch, getSessions (Phase 5)
- admin.js: adminGetProducts, adminCreateProduct, adminUpdateProduct, adminDeleteProduct (Phase 7)

`.env.example` — committed placeholder documenting `SUPABASE_URL` and `SUPABASE_ANON_KEY`; warns against service role key and Gemini API key in any file.

`.gitignore` — excludes `.env`, `node_modules/`, `css/style.css` (generated), `.DS_Store`, `*.log`.
