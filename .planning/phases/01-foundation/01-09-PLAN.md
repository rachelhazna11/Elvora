<objective>
Fix the minor UAT gap from Test 8: the GitHub Actions keep-alive workflow runs only once per week (Monday) instead of daily. A single failed Monday run would leave the Supabase free-tier project paused for up to a week before assessment review.

Purpose: Supabase free-tier projects pause after 1 week of inactivity. The ROADMAP success criterion requires the workflow runs daily. Changing the cron from "0 8 * * 1" (Mondays only) to "0 8 * * *" (every day) ensures the project stays active even if one day's job fails.

Output: Updated .github/workflows/keepalive.yml with daily cron schedule and corrected comment.
</objective>
